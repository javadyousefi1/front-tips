1. What is an IP Address?

An IP address (Internet Protocol address) is a unique numerical label assigned to every device connected to a network. It serves two core purposes: identification (who is this device?) and location addressing (where is this device on the network?).

Without IP addresses, routers would have no way to know where to send data packets. Every request you make from a browser — fetching an HTML file, calling an API, loading an image — travels from your IP to the server's IP and back.

آدرس IP یک شناسه عددی منحصربه‌فرد برای هر دستگاه متصل به شبکه است. دو کار مهم انجام می‌ده: شناسایی دستگاه (کیه؟) و مکان‌یابی روی شبکه (کجاست؟). بدون IP، روترها نمی‌دونن داده رو کجا بفرستن. هر request مرورگر از IP شما به IP سرور و برمی‌گرده.

2. IPv4

IPv4 (Internet Protocol version 4) is the original addressing scheme, introduced in 1981. An IPv4 address is 32 bits long, written as four decimal octets separated by dots — e.g. 192.168.0.1. Each octet ranges from 0–255.

32 bits gives a theoretical maximum of 2³² ≈ 4.3 billion unique addresses. This seemed enormous in 1981, but with the explosion of internet-connected devices we've largely exhausted the pool. Techniques like NAT (Network Address Translation) and CIDR (Classless Inter-Domain Routing) helped stretch the supply, but the long-term fix is IPv6.

IPv4 نسخه اول پروتکل IP است. آدرس ۳۲ بیتی با ۴ عدد ۰-۲۵۵ که با نقطه جدا می‌شن (مثل ۱۹۲.۱۶۸.۰.۱). حداکثر ۴.۳ میلیارد آدرس ممکن داره. این تعداد برای دنیای امروز کافی نبود، به همین خاطر از NAT برای اشتراک‌گذاری IP و CIDR برای تقسیم‌بندی بهتر استفاده می‌شه. راه‌حل اصلی جایگزینی با IPv6 هست.

3. IPv6

IPv6 (Internet Protocol version 6) was standardized in 1998 to solve IPv4 exhaustion. An IPv6 address is 128 bits long, written as eight groups of four hexadecimal digits separated by colons — e.g. 2001:0db8:85a3:0000:0000:8a2e:0370:7334.

128 bits provides 2¹²⁸ ≈ 3.4 × 10³⁸ addresses — enough to give every atom on Earth its own address many times over. IPv6 also removes the need for NAT (every device can get a globally unique address), improves routing efficiency, and has built-in support for auto-configuration (SLAAC).

IPv6 با ۱۲۸ بیت آدرس‌دهی می‌کنه که ۳.۴×۱۰³⁸ آدرس ممکن می‌ده — عملاً نامحدود. به‌صورت ۸ گروه ۴ رقم hex با کولون نوشته می‌شه. مزایا: نیاز به NAT نداره، routing بهتره، پیکربندی خودکار (SLAAC) داره. گروه‌های صفر متوالی رو می‌شه با :: خلاصه کرد (فقط یه بار در آدرس).

3.1 IPv6 Shorthand Rules

Rule 1 Leading zeros in each group can be dropped: 0db8 → db8

Rule 2 One or more consecutive all-zero groups can be replaced with :: (only once per address):

// Full
2001:0db8:0000:0000:0000:0000:0000:0001

// Shortened
2001:db8::1

4. Public IP vs Private IP

4.1 Private IP

A private IP address is used within a local network (home, office, data center). These addresses are defined by RFC 1918 and are not routable on the public internet — routers on the internet will drop packets destined for private ranges.

Every device on your home network gets a private IP from the router (usually via DHCP). Typical ranges: 192.168.x.x for home networks, 10.x.x.x inside cloud VPCs.

آدرس IP خصوصی فقط داخل شبکه محلی معنا داره و در اینترنت عمومی قابل route نیست (RFC 1918). رنج‌ها: ۱۰.۰.۰.۰/۸، ۱۷۲.۱۶.۰.۰/۱۲، ۱۹۲.۱۶۸.۰.۰/۱۶. روتر خانگی به هر دستگاه یک IP خصوصی (مثل ۱۹۲.۱۶۸.۱.x) می‌ده. داخل cloud هم سرورها IP خصوصی دارن.

4.2 Public IP

A public IP address is globally unique and routable on the internet. It's assigned by your ISP (Internet Service Provider) or a cloud provider. When you visit a website, the server sees your public IP — which is the IP of your router, not your laptop.

Public IPs are either static (fixed, common for servers) or dynamic (changes periodically, common for home internet connections). You can see your own public IP at services like ifconfig.me.

IP عمومی به‌صورت جهانی منحصربه‌فرد است و توسط ISP یا ارائه‌دهنده cloud تخصیص می‌یابد. سرور مقصد این IP رو می‌بینه (IP روتر، نه لپ‌تاپ شما). IP عمومی می‌تونه static (ثابت - برای سرورها) یا dynamic (متغیر - برای خانه‌ها) باشه. برای دیدن IP عمومی خودتون می‌تونید از ifconfig.me استفاده کنید.

4.3 NAT — The Bridge Between Private and Public

NAT (Network Address Translation) allows many devices with private IPs to share a single public IP. The router maintains a translation table: when device 192.168.1.5:54321 makes a request, the router rewrites the source to 203.0.113.1:54321 (the public IP), and maps the response back.

NAT (ترجمه آدرس شبکه) به چندین دستگاه با IP خصوصی اجازه می‌ده از یک IP عمومی مشترک استفاده کنن. روتر جدول ترجمه نگه می‌داره: درخواست از ۱۹۲.۱۶۸.۱.۵ با IP عمومی روتر جایگزین می‌شه و جواب به دستگاه اصلی برمی‌گرده. این مکانیزم اصلی‌ترین دلیل کافی بودن IPv4 تا اینجاست.

5. localhost & 127.0.0.1

localhost is a hostname that resolves to 127.0.0.1 (IPv4) or ::1 (IPv6). It refers to the loopback interface — a virtual network interface built into the OS that never sends data to a real network.

When you run a dev server on localhost:3000, the request never leaves your machine. The OS intercepts it and routes it to the local process listening on port 3000. This is why other devices on the same Wi-Fi can't access localhost on your computer — it's local-only by design.

localhost یک hostname هست که به ۱۲۷.۰.۰.۱ (IPv4) یا ::1 (IPv6) ترجمه می‌شه. این آدرس loopback interface است — یک اینترفیس مجازی که داده رو هرگز به شبکه واقعی نمی‌فرسته. وقتی dev server روی localhost:3000 اجرا می‌کنی، درخواست از ماشین خارج نمی‌شه. به همین دلیل دستگاه‌های دیگه روی همان Wi-Fi نمی‌تونن به localhost دسترسی داشته باشن — عمداً فقط محلیه.
Dev Tip To let other devices on the same network reach your dev server, bind it to 0.0.0.0 instead of localhost. Then access it via your machine's LAN IP (e.g., 192.168.1.5:3000).

5.1 The Loopback Flow

جریان loopback: مرورگر درخواست به localhost:3000 می‌فرسته → سیستم‌عامل آن رو intercept می‌کنه → داده از طریق اینترفیس مجازی ۱۲۷.۰.۰.۱ به پروسه dev server روی پورت ۳۰۰۰ می‌رسه. هیچ داده‌ای از کارت شبکه واقعی خارج نمی‌شه.

6. What Does 0.0.0.0 Mean?

0.0.0.0 is a special-purpose address with two distinct meanings depending on context:

  • As a bind address (server context): "Listen on all available network interfaces." A server bound to 0.0.0.0:8080 accepts connections arriving on any IP the machine has — loopback, LAN, public interface, etc.
  • As a route/source address: "This host, on this network" — used before the device has been assigned an IP (e.g., during DHCP discovery).
۰.۰.۰.۰ دو معنا داره: ۱) در context سرور (bind address): «روی همه اینترفیس‌های شبکه گوش بده». سرور bind شده به ۰.۰.۰.۰ هم از localhost قابل دسترسه هم از شبکه LAN. ۲) در context routing/DHCP: «این دستگاه قبل از گرفتن IP». در مقابل، localhost/127.0.0.1 فقط loopback هست.

فرق bind به ۱۲۷.۰.۰.۱ vs ۰.۰.۰.۰: bind به ۱۲۷.۰.۰.۱ فقط از خود ماشین قابل دسترسه. bind به ۰.۰.۰.۰ از localhost و از شبکه LAN هم قابل دسترسه. دسترسی از اینترنت به firewall بستگی داره. مثلاً Vite با --host به ۰.۰.۰.۰ bind می‌کنه تا از موبایل روی همان Wi-Fi قابل تست باشه. 
# Vite: bind to all interfaces so teammates can test on their phone
vite --host

# Node.js: same effect manually
server.listen(3000, '0.0.0.0')

# Check what your machine is listening on (macOS/Linux)
lsof -i :3000
Warning Binding to 0.0.0.0 in a cloud environment without a firewall means your service is potentially reachable from the public internet. Always configure security groups / firewall rules.
